Compiler Backends

Decretum compiles to 84 targets covering every major architecture family from 4 bit microprocessors to 64 qubit quantum circuits. The syntax is the same across all targets, but the programs themselves are target specific you use each architecture's own registers, instructions, and addressing modes. Each backend has its own instruction encoding, register model, and output format, and runs a peephole optimization pass. Some backends use a two pass assembly model with label resolution; others use a single pass model with in place branch patching. All backends support high level if, elif, else, endif, while, and endwhile control flow constructs.

How to Compile

Use compile-arch which detects the target from the source file and dispatches to the right backend automatically:

decretum compile-arch program.dcrt --out-file output.bin

Or use the target specific command directly:

decretum compile-riscv program.dcrt --out-file output.bin

Optimization Passes

Every backend runs a peephole optimization pass before final encoding. The pass collapses consecutive NOPs into one, removes dead code after unconditional branches and returns, fills delay slots for MIPS, SPARC, and OpenRISC targets, eliminates SPARC save and restore window pairs, and simplifies PowerPC CR logical operations. A byte level pass then strips trailing filler bytes, deduplicates consecutive filler bytes, and removes trailing zeros after all other passes complete.

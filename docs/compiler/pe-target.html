Windows PE Target

Build Flow

  1. Compile Decretum source into portable bytecode using the same pipeline as compile-bytecode
  2. Embed the bytecode payload into the runtime executable by appending it after the PE sections
  3. The runtime executable is a prebuilt binary from src/bin/dcrt_rt.rs that scans itself for the embedded bytecode at launch

The PE runtime executes the embedded Decretum bytecode entry event when launched. It reads its own executable file, locates the bytecode payload via a 16 byte magic signature, and passes it to the BytecodeRuntime interpreter. The magic signature uses the byte sequence {0xDE, 0xC0, ...} with each byte offset by +1 from the standard DCB header to prevent false matches.

Usage

cargo run -- compile-pe examples/compute_pe.dcrt --out-file build/compute_pe.exe

The output is a standalone Windows executable that runs the compiled Decretum program. No external dependencies, DLLs, or runtime installations are required.

Runtime Profiles

Two runtime profiles exist. The lightweight profile supports standard arithmetic, I/O, control flow, and string operations. The full profile adds support for the compile_decretum opcode, enabling self compilation at runtime. The full profile is enabled by the compiler-op Cargo feature.

Notes

No NASM or external assembler is required for this path. The PE executable is self contained with all runtime logic baked into the binary. The dcrt_rt runtime stub can also be compiled as a standalone binary that loads .dcb files from disk at runtime.

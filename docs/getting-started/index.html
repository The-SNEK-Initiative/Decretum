Getting Started

All commands use cargo run --bin decretum -- from the project root. You can also build a release binary with cargo build --release --bin decretum and run it directly as decretum.

1. Validate Source

cargo run --bin decretum -- validate examples/full_kernel.dcrt

The parser checks for correct syntax, valid target names, and proper block structure. It reports any errors with line numbers and descriptions.

2. Build a Bootable BIOS Image

cargo run --bin decretum -- compile-bootimg examples/full_kernel.dcrt --out-file build/full_kernel.img

Outputs a 512 byte boot sector plus a stage2 kernel payload. The boot sector uses CHS disk reads through INT 13h to load the kernel. Write the image to USB with Rufus in image mode and boot in BIOS or CSM mode.

3. Build Portable Bytecode

cargo run --bin decretum -- compile-bytecode examples/compute_pe.dcrt --out-file build/compute_pe.dcb

Bytecode files use the .dcb format and run through the built in BytecodeRuntime interpreter. The runtime supports 51 opcodes, 16 registers, string operations, and a 320 bit PRNG.

4. Run Bytecode Programs

cargo run --bin decretum -- compile-bytecode .github/examples/random.dcrt --out-file build/random.dcb

You can run .dcb files through the IDE which embeds the portable runtime, or compile to a PE executable that includes the runtime stub.

5. Compile to Any Architecture

The compile-arch command detects the target from the source file and dispatches to the right backend automatically. Change the target directive in the source to compile for any of the 84 supported architectures:

cargo run --bin decretum -- compile-arch .github/examples/compute.dcrt --out-file build/compute_mips.bin

6. Build a Native Windows PE

cargo run --bin decretum -- compile-pe examples/compute_pe.dcrt --out-file build/compute_pe.exe

The PE executable embeds the portable bytecode alongside a runtime stub. When launched, the stub finds the embedded bytecode via a magic signature and runs it through the interpreter.

7. Compile a MIPS Binary

echo target mips > test.dcrt
echo entry main >> test.dcrt
echo event main: >> test.dcrt
echo     li r1, 42 >> test.dcrt
echo     ret >> test.dcrt
cargo run --bin decretum -- compile-arch test.dcrt --out-file build/test_mips.bin
hexdump -C build/test_mips.bin

9. Compile a Quantum Circuit

echo target quantum8 > bell.dcrt
echo entry main >> bell.dcrt
echo event main: >> bell.dcrt
echo     h q0 >> bell.dcrt
echo     cx q0, q1 >> bell.dcrt
echo     measure q0, c0 >> bell.dcrt
echo     ret >> bell.dcrt
cargo run --bin decretum -- compile-arch bell.dcrt --out-file build/bell.qbin

The quantum backend supports 15 gate types including Hadamard, Pauli X/Y/Z, phase (S and T), controlled-X, Toffoli, rotation gates, measurement, and barriers. The 64 qubit variant includes circuit optimization and OpenQASM 2.0 export.

10. Compile for ARM Cortex-M

echo target armcm > blink.dcrt
echo entry main >> blink.dcrt
echo event main: >> blink.dcrt
echo     mov r0, 1 >> blink.dcrt
echo     ret >> blink.dcrt
cargo run --bin decretum -- compile-arch blink.dcrt --out-file build/blink.bin

The Cortex-M backend generates a raw binary with a vector table at address 0x0000 for direct flashing to microcontrollers.

10. Launch the IDE

cargo run --bin ide

The IDE provides a full code editor with syntax highlighting, a file explorer, inline compilation, runtime execution, and a built in documentation viewer. It can run .dcb bytecode files directly and launch .exe files as subprocesses.

12. Write BIOS Image to USB

Use Rufus to write build/full_kernel.img in image mode. Boot the target machine in BIOS or CSM compatibility mode.

12. Run Tests

cargo test

All integration tests and unit tests must pass. The test suite covers parser validation, bytecode compilation and execution, bytecode file I/O, BIOS boot image generation, quantum circuit encoding, byte level optimization, and peephole optimization.

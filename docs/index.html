Decretum DCRT

Decretum DCRT is an event driven low level language with 84 compiler targets. The same syntax works across all of them, but the code you write is still target specific: a program written for MIPS will not compile for quantum without changes. You write for each architecture using the same event and proc block structure, data declarations, and control flow constructs, but you use that architecture's registers, instructions, and calling conventions.

Design Goals

  • Assembler level control with clearer structure than raw assembly
  • Event blocks as first class control flow units
  • High level constructs like if and while that compile to native jumps
  • Portable bytecode path for host executables on Windows and Linux
  • Direct machine code path for every major architecture family in computing history
  • String operations, assertions, random generation, and math built into the language
  • Zero external assembler or linker dependencies for any target

Architecture Targets

Decretum compiles to 84 targets spanning 19 architecture families: x86 (8 variants), Intel evolution (3), ARM (6), RISC-V (3), classic RISC (7), FPGA soft cores (5), academic or educational (3), 8 bit CPU classics (5), 8 bit MCUs (5), 16 bit MCUs (6), 32 bit MCU or RISC (3), DSPs (3), Hitachi, 68k, and console (4), PDP, VAX, and HP (4), mainframe and vintage (4), VLIW (3), military and aerospace (2), Soviet and exotic (6), and special purpose plus quantum (5). Every backend has its own instruction encoding, register model, and peephole optimizer; some use a two pass assembly model with label resolution, others a single pass model with in place branch patching. All backends support high level if/elif/else/endif/while/endwhile control flow constructs.

See the full backend list for details on every target, its CLI command, output format, and architecture notes.

Target Coverage can be found in Target Constraints

Internal Architecture

The Decretum compiler is organized into a parser frontend, a portable bytecode backend, and 84 direct machine code backends. The parser tokenizes .dcrt source into a Program structure with target, entry event, data declarations, and code blocks. Code blocks contain raw instruction lines that each backend interprets using its own lower function.

The portable backend compiles to a 51 opcode bytecode format executed by a register based VM with 16 registers, a memory map, string storage, and a 320 bit ARX PRNG called DCRNG. The direct backends each encode instructions to binary and write the target output format, using either a two pass model with label maps or a single pass model with in place branch patching. A shared peephole optimizer collapses NOPs and removes dead code, and a byte level optimizer strips trailing fill bytes.

The IDE is built with egui and eframe. It embeds all documentation as static HTML strings via the docs_data module and can run embedded bytecode payloads through the portable runtime.

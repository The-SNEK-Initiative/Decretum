Syntax

Required Headers

Every Decretum source file must specify a target architecture and an entry event:

target portable
entry boot

Data Declarations

data banner = "hello\0"
byte flag = 1
word mode = 2
dword count = 64
qword ticks = 0
buffer input 256

Data declarations are global and accessible by name from any block. The buffer type allocates a zeroed block of memory. String data uses escape sequences: \n for newline, \r for carriage return, \t for tab, \\ for backslash, \" for literal quote, and \0 for NUL.

Code Blocks

event boot:
    emit scheduler
    ret

proc scheduler:
    mov rax, 42
    ret

Block body lines must be indented. event blocks are dispatchable through emit or set as the entry point. proc blocks are called through call and return with ret.

Control Flow Constructs

High level block constructs that expand to compare and jump sequences. Supported on every backend:

if r0
    ; executes when r0 is not zero
elif r1
    ; executes when r0 is zero and r1 is not zero
else
    ; executes when both r0 and r1 are zero
endif
while r0
    ; repeats while r0 is not zero
    dec r0
endwhile

The if block opens a conditional chain. elif adds additional conditions. else catches the fall through. endif closes the chain. The while block loops until its condition becomes zero. All constructs nest correctly and the compiler rejects unclosed blocks.

Labels

.loop:
    dec rcx
    jnz .loop

Local labels starting with a dot are scoped to their containing block. The assembler qualifies them with the block name to prevent name collisions.

Registers and Operands

Register names depend on the target architecture. For x86 targets use ax, bx, cx, dx, si, di, sp, bp, and extended forms like eax, rax. For RISC targets use numbered registers like r0 through r31. For quantum targets use qubit references like q0 through q63 and classical bit references like c0 through c63. Immediate values can be decimal or hexadecimal with the 0x prefix.

Comments

; this is a comment
mov ax, 1 ; inline comment

The semicolon starts a comment that extends to the end of the line. Comments inside quoted strings are treated as literal text. Multi statement lines are not supported the semicolon is reserved for comments only.

Target Directive

The target directive accepts any of the 84 supported architecture names. The compiler validates the target at parse time and reports an error for unknown targets. The full list of target names is available on the Backends page.
